# Product Requirements

## 1. Problem
Merchants need a simple way to increase revenue per order through upsell and cross-sell offers, but existing workflows are often too complex, unclear, or hard to measure.

## 2. Goals
- Enable merchants to create upsell offers easily
- Help merchants activate offers quickly
- Provide visibility into offer performance
- Improve AOV and conversion rate

## 3. Users
### Primary
- Shopify merchants
- Store owners
- E-commerce managers

### Secondary
- Agencies
- Store operators

## 4. Core features

### 4.1 Onboarding
Merchant should be guided through:
- app introduction
- value proposition
- first setup steps
- creating first offer

### 4.2 Dashboard
Merchant can view:
- total upsell revenue
- impressions
- clicks
- conversions
- top performing offers
- setup progress
- recommended next actions

### 4.3 Offer list
Merchant can:
- view all offers
- search and filter offers
- see offer status
- duplicate offer
- edit offer
- enable/disable offer

### 4.4 Offer placement
Supported placements in MVP:
- product page
- cart
- post purchase page
- thankyou page
- checkout page

### 4.5 Offer builder
Merchant can:
- choose page upsell
- create an offer
- define offer name
- select offer type: Cross-sell/ Frequently bought together/ Product add-on
- choose trigger placement
- choose target products or conditions: All products/ Specific products/ Products in specific collections
- configure offer content: Specific products/ Automated recommendations
- setup discount: Percentage/ Fixed amount
- works with other offers: Priority, Stop on match
- enable/disable the offer

### 4.6 Analytics
Merchant can view:
- impressions
- clicks
- click-through rate
- conversion count
- conversion rate
- revenue generated by offer

### 4.7 Settings
Merchant can:
- configure app general settings
- manage widget behavior
- control display preferences

## 5. Business rules
- Merchant can create multiple offers
- Offer must have a name before saving
- Offer must have at least one placement
- Offer can be enabled only when minimum required configuration is complete
- Disabled offers must not display on storefront
- Analytics only count active offer interactions
- Offer metrics should be attributable to specific offers

## 6. Functional requirements

### FR1. Merchant onboarding
- System must show a welcome/onboarding experience for new users
- System must guide merchant to create the first offer

### FR2. Offer creation
- System must allow merchant to create a new offer
- System must validate required fields before save
- System must allow draft save

### FR3. Offer management
- System must show all created offers
- System must support enable/disable status
- System must support edit and duplicate actions

### FR4. Dashboard
- System must display summary metrics
- System must show recent or top-performing offers
- System must surface onboarding/setup progress for new users

### FR5. Analytics
- System must record offer impressions
- System must record offer clicks
- System must record offer conversions
- System must display revenue generated by offer

## 7. Non-functional requirements
- Admin pages should load quickly and be easy to scan
- UI should follow Shopify admin expectations
- Terminology should be simple for non-technical users
- System should support future scalability for advanced targeting and experimentation

## 8. MVP acceptance criteria

### Onboarding
- Merchant can understand what the app does
- Merchant can navigate to create the first offer

### Dashboard
- Merchant sees at least core KPIs clearly
- Merchant sees recommended next step

### Offer list
- Merchant can view, search, edit, duplicate, and disable offers

### Offer builder
- Merchant can create and save an offer successfully
- Merchant can activate an offer when required inputs are complete

### Analytics
- Merchant can see core offer performance metrics

## 9. Risks / open questions
- What level of analytics granularity is needed in phase 1?
- How should empty state and first-time experience be handled to maximize adoption?
